Technical Guide · PocketBook

Custom Dictionary
for PocketBook

Convert StarDict or Kaikki (Wiktionary) dictionaries to PocketBook's native format using a modular Java toolchain.

StarDict / Kaikki → TSV → XDXF → .dic Linux · macOS · Windows Java 17+
Formats & Data Sources

The pipeline passes data through several formats, each with a distinct role. Understanding them helps troubleshoot issues and adapt the toolchain to new sources.

Source · StarDict
StarDict.ifo .idx .dict.dz
A widely supported open dictionary format used by GoldenDict, Aard2, and others. A dictionary consists of three files: .ifo (metadata), .idx (binary word index with byte offsets), and .dict.dz (gzip-compressed definitions). Definitions are typically HTML. Source: reader-dict.com.
Source · Kaikki / Wiktionary
Kaikki JSONL.jsonl .jsonl.gz
Community-maintained Wiktionary data extracted and published by kaikki.org, updated weekly. Each line is a self-contained JSON object for one headword containing senses, IPA pronunciation, etymology, gender, plural forms, domain labels, usage examples, and compound expressions. Richer and more current than most packaged dictionaries.
Intermediate
Tabfile / TSV.tsv
Plain-text tab-separated values: one entry per line, word<TAB>definition. The common interchange format between all source converters and the XDXF writer. Useful for inspection, diffing, and debugging — keep it on disk as a build artefact. Use out=- / in=- to pipe directly between tools without writing to disk. Definitions may contain HTML.
Intermediate
XDXF.xdxf
XML-based dictionary format consumed by xdxf-2-pbdic. Each entry is an <ar> element with a <k> headword and an HTML definition body. Strict XML — tag attributes are stripped, unknown entities are escaped. Output is validated with Java's SAX parser before the next step.
Output · PocketBook native
PocketBook Dictionary.dic
Proprietary binary format consumed by PocketBook firmware. The format has been reverse-engineered and is produced natively by xdxf-2-pbdic, a pure-Java tool included in this toolchain. No Wine or closed-source tools required. Conversion requires per-language collation, keyboard, and morphem files from LanguageFilesPocketbookConverter by Markismus.

Steps
1
Download the dictionary source

Choose your source. StarDict gives a ready-made packaged dictionary from reader-dict.com. Kaikki gives richer, community-maintained Wiktionary data updated weekly — pronunciation, etymology, domain labels, examples, and compound expressions included.

Go to reader-dict.com and download the StarDict package (.zip). Extract it — you will get .ifo, .idx, and .dict.dz files.

↓
reader-dict — Portuguese (PT-PT) reader-dict.com/pt/download/pt

Download the JSONL dump from kaikki.org. Each line is one JSON object representing a headword with all its linguistic data. The file is updated weekly from the English Wiktionary. A .jsonl.gz compressed version is also accepted by kaikki-2-tab.

↓
Kaikki — Portuguese JSONL dump kaikki.org/dictionary/downloads/pt/pt-extract.jsonl
Entry coverage: part of speech, gender, plural forms, Portugal IPA pronunciation, numbered senses with domain labels and usage examples, etymology, and compound expressions as standalone lookup entries.
2
Download language files

xdxf-2-pbdic requires per-language collation, keyboard, and morphem files to sort and index the dictionary correctly. These are maintained separately by Markismus and are not included in this repo — download them manually and place them in your working directory. 

# Download the zip from GitHub
wget https://github.com/Markismus/LanguageFilesPocketbookConverter/archive/refs/heads/master.zip -O langfiles.zip
unzip langfiles.zip

# Copy the folder for your target language into lang/
mkdir -p lang/
cp -r LanguageFilesPocketbookConverter-master/pt lang/pt
Files needed per language: collates.txt, keyboard.txt, morphems.txt — all three must be present in the language folder. The folder name must match the language code passed to --lang (e.g. lang/pt for -l pt).
License note: the LanguageFilesPocketbookConverter repository carries no explicit license. It is publicly available on GitHub but all rights are reserved by the author. Use it for personal dictionary conversion; do not redistribute the files in your own repo or package.
↓
LanguageFilesPocketbookConverter — all languages (zip) github.com/Markismus/LanguageFilesPocketbookConverter
3
Set up folder structure

Place the conversion tools in linux/, source data in data/, and the language files you downloaded in lang/. Generated .tsv, .xdxf, and .dic files land in data/out/.

# Expected folder structure
linux/
  stardict-2-tab        # StarDict binary → TSV
  kaikki-2-tab          # Kaikki JSONL → TSV
  tab-2-xdxf            # TSV → XDXF (both pipelines)
  xdxf-2-pbdic          # XDXF → .dic (pure Java)
  check-pbdic           # validate .dic against .xdxf
data/
  stardict/
    dict-pt-pt/         # StarDict source (if using StarDict)
      dict-data.ifo
      dict-data.idx
      dict-data.dict.dz
  kaikki/               # Kaikki source (if using Kaikki)
    pt-extract.jsonl    # or .jsonl.gz
  out/                  # all generated files
    *.tsv  *.xdxf  *.dic
lang/                # NOT in repo — download separately (see step 2)
  pt/
    collates.txt  keyboard.txt  morphems.txt
confs/kiakki2xdxf-pten-pt.config
confs/kiakki2xdxf-pt-pt.config
confs/stardict2xdxf-pt-pt.config
Language files go in lang/<lang>/ — this folder is not distributed with the toolchain. See step 2 for how to obtain them.
4
Convert to TSV and XDXF

All tools are self-contained Java source scripts — no compilation step needed. The two-stage pipeline produces an intermediate .tsv file on disk, useful for inspection and debugging before committing to the final XDXF conversion.

stardict-2-tab reads the StarDict binary files and writes a TSV. tab-2-xdxf converts the TSV to XDXF and validates the output XML.

# Give execute permission (once)
chmod +x linux/stardict-2-tab linux/kaikki-2-tab linux/tab-2-xdxf linux/xdxf-2-pbdic linux/check-pbdic

# Step 1 — StarDict → TSV
linux/stardict-2-tab -i data/dict-pt-pt/dict-data.ifo -o data/out/dict-pt-pt.tsv

# Step 2 — TSV → XDXF
linux/tab-2-xdxf -i data/out/dict-pt-pt.tsv -o data/out/dict-pt-pt.xdxf -l pt -n "Dicionário PT-PT"

# Step 3 — XDXF → .dic
linux/xdxf-2-pbdic -i data/out/dict-pt-pt.xdxf -o data/out/dict-pt-pt.dic -l pt -D lang/pt
Language codes: accepts ISO 639-1 (pt) or ISO 639-2 (POR) — two-letter codes are converted automatically.
JDK required: the shebang launcher compiles on the fly; a full JDK is needed, not just the JRE.

kaikki-2-tab parses the Kaikki JSONL and writes a TSV filtered by language code. tab-2-xdxf then converts it to XDXF. Both .jsonl and .jsonl.gz are accepted as input.

# Give execute permission (once)
chmod +x linux/stardict-2-tab linux/kaikki-2-tab linux/tab-2-xdxf linux/xdxf-2-pbdic linux/check-pbdic

# Step 1 — Kaikki JSONL → TSV (single language, expressions as separate entries only)
linux/kaikki-2-tab -i data/kaikki/pt-extract.jsonl -o data/out/kaikki-pt.tsv -l pt -e SEPARATE

# Step 1 — multiple languages
linux/kaikki-2-tab -i data/kaikki/pt-extract.jsonl -o data/out/kaikki-pt-en.tsv -l pt,en -e SEPARATE

# Step 2 — TSV → XDXF
linux/tab-2-xdxf -i data/out/kaikki-pt.tsv -o data/out/kaikki-pt.xdxf -l pt -n "Dicionário PT (Kaikki)"

# Step 3 — XDXF → .dic
linux/xdxf-2-pbdic -i data/out/kaikki-pt.xdxf -o data/out/kaikki-pt.dic -l pt -D lang/pt
Kaikki data is updated weekly — re-download pt-extract.jsonl periodically to keep your dictionary current.
--embedded-defs controls how compound expressions are emitted: SEPARATE (standalone lookup entries only), KEEP (embedded inside the parent entry only), or BOTH (default). Use SEPARATE to avoid artefact headwords from Kaikki expression data.

Each conversion has its own config file. All tools read it via -c and use their own namespace — CLI arguments always override config values.

Property keys are prefixed with the tool name, dashes removed. Setting out=- or in=- enables piping directly from the config.

# confs/kiakki2xdxf-pten-pt.config — Kaikki PT+EN, piped to file

kaikki2tab.in=data/kaikki/pt-extract.jsonl.gz
kaikki2tab.out=-
kaikki2tab.lang=pt,en
kaikki2tab.embedded-defs=SEPARATE

tab2xdxf.in=-
tab2xdxf.out=data/out/kaikki-pten-pt.xdxf
tab2xdxf.lang=pt,en
tab2xdxf.name=Wiktionary.org PT+EN->PT (Kaikki)
# confs/stardict2xdxf-pt-pt.config — StarDict, piped to file

stardict2tab.in=data/stardict/dict-pt-pt/dict-data.ifo
stardict2tab.out=-

tab2xdxf.in=-
tab2xdxf.out=data/out/readerdict-pt-pt.xdxf
tab2xdxf.lang=pt
tab2xdxf.name=Wiktionário PT->PT (ReaderDict)
# Run with the chain scripts
linux/chain-kaikki-2-xdxf   confs/kiakki2xdxf-pten-pt.config
linux/chain-stardict-2-xdxf confs/stardict2xdxf-pt-pt.config

# Or run tools individually
linux/kaikki-2-tab  -c confs/kiakki2xdxf-pten-pt.config
linux/tab-2-xdxf   -c confs/kiakki2xdxf-pten-pt.config

# CLI override — different language
linux/kaikki-2-tab -c confs/kiakki2xdxf-pt-pt.config -l pt,en -o - | linux/tab-2-xdxf -i - -c confs/kiakki2xdxf-pt-pt.config
Namespaces: stardict-2-tab → stardict2tab.*, kaikki-2-tab → kaikki2tab.*, tab-2-xdxf → tab2xdxf.*, xdxf-2-pbdic → xdxf2pcdic.*, check-pbdic → checkpbdic.*.
Piping: all progress messages go to stderr, so only data flows through the pipe. XML validation runs automatically even in pipeline mode (via a temporary file) — no flags needed.
5
Convert XDXF to .dic

xdxf-2-pbdic is a pure-Java tool included in this toolchain — no Wine or closed-source dependencies required. 

# StarDict pipeline
linux/xdxf-2-pbdic -i data/out/dict-pt-pt.xdxf -o data/out/dict-pt-pt.dic -l pt -D lang/pt -j false
linux/xdxf-2-pbdic -i data/out/kaikki-pt.xdxf -o data/out/kaikki-pt.dic -l pt -D lang/pt -j true

# Full chain piped into xdxf-2-pbdic (shared config)
linux/chain-kaikki-2-xdxf confs/kiakki2xdxf-pt-pt.config
linux/xdxf-2-pbdic -c confs/kiakki2xdxf-pt-pt.config
Language files: the -D lang/pt flag points to a folder containing collates.txt, keyboard.txt, and morphems.txt for your target language. Download from LanguageFilesPocketbookConverter by Markismus.
--join (required): controls how headwords that match case-insensitively are handled. -j true merges them into one entry (lowercase headword kept, blank line between definitions) — recommended for Kaikki sources. Entries whose definitions are also equal (case-insensitive) are considered exact duplicates and discarded. -j false keeps entries with different casing separate (e.g. pessoas and Pessoas) — recommended for StarDict sources. Only byte-identical duplicates (same headword exact case + same definition bytes) are discarded. Set via config as xdxf2pcdic.join=true.
Output: the .dic file is written to data/out/.
5b
Validate (optional)

check-pbdic compares the generated .dic against the source XDXF, checking entry count, headword presence, sort order, and a sampled spot-check of definitions. 

linux/check-pbdic -x data/out/kaikki-pt.xdxf -d data/out/kaikki-pt.dic -l pt -n 50
Exits with code 0 on success, 1 if any check fails — suitable for use in a build script or CI pipeline.
6
Copy to PocketBook

Connect your PocketBook via USB and copy the generated .dic file to the dictionaries folder on the device. 

# StarDict source
cp data/out/dict-pt-pt.dic /media/POCKETBOOK/system/dictionaries/

# Kaikki source
cp data/out/kaikki-pt.dic /media/POCKETBOOK/system/dictionaries/
The mount point /media/POCKETBOOK/ may differ depending on your system and distro — check where the device is mounted before copying. Safely eject the device after copying; a restart may be needed for the dictionary to appear in the language settings.
Legacy
dict-2-xdxf — deprecated. Earlier versions of this guide used a single dict-2-xdxf tool that handled both StarDict and tabfile input via a --format flag. It has been superseded by the modular toolchain above (stardict-2-tab, kaikki-2-tab, tab-2-xdxf), which separates reading and writing concerns and converges on a common TSV intermediate format. dict-2-xdxf remains functional for single-step StarDict or tabfile conversion, but will not receive further updates.
converter.exe — deprecated. Earlier versions of this guide described converter.exe (a patched Windows binary run via Wine) as an alternative final step for producing .dic files from XDXF. It has been superseded by xdxf-2-pbdic, a pure-Java tool included in this toolchain that requires no Wine, no Windows, and no closed-source dependencies. xdxf-2-pbdic produces output verified to be compatible with PocketBook firmware and is the only supported conversion path going forward.
